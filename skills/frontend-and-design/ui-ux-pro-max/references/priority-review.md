# Priority Review

Use this order whenever you review or improve an interface.

## 1. Accessibility and readability

Check first:

- text contrast is readable
- body text is not undersized
- interactive elements have visible focus states
- forms have labels and understandable errors
- icon-only controls have clear labels or context

If this layer is weak, stop here and fix it before style work.

## 2. Interaction and touch safety

Check next:

- tap targets are large enough
- controls are not too tightly packed
- primary actions are obvious
- loading, disabled, success, and error states are visible
- hover is not the only way to discover functionality

## 3. Layout and responsiveness

Check:

- mobile layout does not horizontally scroll
- main content is clearly prioritized on small screens
- fixed headers, drawers, and bottom bars do not trap content
- sections have a consistent spacing rhythm
- long lines and dense grids are constrained on large screens

## 4. Hierarchy and comprehension

Check:

- the page has one dominant action or focus
- headings, body, metadata, and helper text read at different levels
- cards and panels are grouped by purpose, not only by decoration
- visual weight matches business importance

## 5. Visual system consistency

Check:

- color choices feel intentional, not random
- corner radius and shadow language are consistent
- typography scale feels systematic
- repeated components share the same state model

## 6. Performance-sensitive UI choices

Check:

- image-heavy views reserve space and avoid jumping
- large lists are not visually overloaded
- effects like blur, parallax, and animation are not overused

## 7. Motion and polish

Check last:

- transitions help explain state changes
- durations feel responsive
- reduced-motion needs are respected
- decorative motion is minimal and purposeful

## Scoring shortcut

If a UI feels bad but the reason is unclear, ask:

1. Can I read it?
2. Can I use it quickly?
3. Can I understand what matters?
4. Does it feel like one system?

That usually reveals the real problem faster than tweaking styles.
